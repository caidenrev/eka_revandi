Masa transisi mental yang paling membingungkan bagi pemula di bidang pengembangan *software* biasanya terjadi setelah mereka menyelesaikan aplikasi mereka di *localhost:3000*. Pertanyaan pertamanya adalah: *"Terus bagaimana caranya agar link aplikasi ini bisa dibuka teman saya lewat HP-nya?"*

Jawabannya adalah memindahkan kode (deployment) tersebut dari komputer lokal milikmu ke "Cloud". Masalahnya, begitu membuka dashboard cloud provider mana pun (misal AWS Console), yang ada malah antarmuka penuh list membingungkan, ratusan layanan dengan singkatan aneh (IAM, EC2, ECS, VPC, S3). Bagaimana kita memecah kompleksitas ini?

Berikut adalah alur progresif (dari gampang hingga kompleks) yang akan memandu Anda:

## Tahap 1: Memahami "Object Storage" untuk Hosting Web Statis

Sebagian besar proyek web pertama developer pemula adalah website *Statis* (seperti portfolio sekedar HTML/CSS murni, atau React Single Page Application yang sudah ter-compile).

Jika aplikasi Anda bersifat front-end statis tanpa ada logic API belakang dan *database*, tidak perlu menyewa layanan *server mahal*. Gunakanlah fitur yang disebut *"Object Storage"* (Layanan berbasis folder penyimpanan online).

- **AWS Service Pilihan:** Amazon S3. 
  Amazon S3 sangat brilian karena kita cukup *upload* folder statis HTML/CSS layaknya *Google Drive*. Jika fitur **"Static Website Hosting"** diaktifkan, Amazon S3 akan mengubah bucket penyimpananmu menjadi host website dengan performa tanpa jeda dan tangguh (karena disalurkan lewat CDN). Biaya menggunakan ini hanyalah sepersekian *dollar* (atau gratis total). 

## Tahap 2: Menangani Backend melalui PaaS

Aplikasi Anda naik kelas. Ia sekarang bukan lagi web statis, tetapi web *dinamik* yang menjalankan proses Node.js, Next.js *Server-Side Rendering*, atau Flask (Python). Anda kali ini mendadak *"butuh komputasi"* – CPU untuk menghitung proses berulang tanpa henti.

Daripada meloncat memakai Virtual Machine mentah (yang mewajibkan kemampuan terminal linux mumpuni), mari lirik PaaS (Platform as a Service).

- **AWS Service Pilihan:** AWS Elastic Beanstalk atau AWS App Runner.
  Dengan App Runner, Anda bahkan cukup menghubungkan GitHub ke AWS. Saat Anda melakukan `git push`, AWS akan memindai kode Anda, mencari ekstensi bahasa, menyusun (*build*) infrastrukturnya, dan langsung menjalankannya (*deploy*) memberikan sebuah link aktif. Sungguh "Developer Friendly"!

## Tahap 3: Virtualisasi Modern Menggunakan Docker (Containers)

Ini adalah tahapan The Real Cloud Engineer. Kode yang berjalan semulus sutra di laptopmu mungkin akan tiba-tiba penuh *error file-path* ketika dijalankan secara telanjang di server Linux milik perusahaan. Mengapa? Karena versi *Environment Variables* atau versi frameworknya tidak sinkron!

Maka muncullah standarisasi **DOCKER**. Docker *membungkus* kodemu, library-mu, beserta versi OS yang dibutuhkan kodemu ke dalam sebuah "Container" layaknya peti kemas kontainer kapal. Karena menggunakan peti kemas dengan standar keamanan industri perkapalan, maka siapapun yang mengantarkan peti kemasmu, keamanannya terjamin.

- **AWS Service Pilihan:** Amazon ECS (Elastic Container Service).
  Saat Anda membangun file kontainer Anda, Anda sekedar mengunggahnya (upload) ke AWS. Keajaiban ECS akan otomatis mendistribusikan ratusan kontainermu menduplikasi *node* sehingga menahan hantaman ratusan ribu pengunjung bukan masalah.

## Tahap 4: Mengasingkan Database (Cloud Managed DB)

Salah satu dosa terbesar arsitektur pemula: menaruh aplikasi dan server *Database* *(PostgreSQL/MySQL/MongoDB)* berdampingan pada satu "Komputer" virtual yang sama.

Jika trafiknya meledak dan server tersebut *overloaded*, matilah segalanya. Oleh karenanya di dunia Cloud, *Database* harus dibuat di *layer* mesin tersendiri yang berdiri *Independent*.

- **AWS Service Pilihan:** Amazon RDS (Relational Database Service).
  Anda bebas menentukan seberapa besar kapasitas mesin databasenya. Dan yang lebih mengagumkan: RDS sanggup untuk mencadangkan *(backup)* serta mengatur update celah celah keamanan *(Patching)* secara mandiri. Sang Engineer kini bisa lebih nyenyak tidur di akhir pekan.

Jadi, mulailah langkah Cloud perjalananmu. *Commit code*-mu hari ini, tentukan mana solusi yang tepat!
